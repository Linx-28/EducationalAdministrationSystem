# 安全策略

## 报告漏洞

如果你发现安全漏洞，请**不要**公开报告。请通过以下方式联系我们：

1. 在 [Issues](https://github.com/Linx-28/EducationalAdministrationSystem/issues) 页面创建一个私密报告（如果可用）
2. 或者通过 GitHub 的安全漏洞报告功能

## 安全最佳实践

### 数据库安全

- 不要在代码中硬编码数据库密码
- 使用环境变量或配置文件存储敏感信息
- 确保 `db.properties` 不会被提交到版本控制

### 部署安全

- 使用 HTTPS
- 定期更新依赖库
- 限制数据库用户权限
- 使用强密码

### 代码安全

- 对用户输入进行验证
- 使用参数化查询防止 SQL 注入
- 对输出进行编码防止 XSS 攻击
- 实施适当的访问控制
